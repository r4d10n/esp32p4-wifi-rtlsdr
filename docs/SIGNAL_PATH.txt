│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │ STAGE 4: FM Discriminator                                         │   │
│  │ fm_disc_process(state, iq_decim, audio, pairs)                    │   │
│  │ (pie_fm_discriminator.c — Method A: Polynomial corrected atan2)   │   │
│  │                                                                   │   │
│  │ For each IQ pair k:                                               │   │
│  │   cross = I[k]·Q[k-1] - Q[k]·I[k-1]  (conjugate multiply)      │   │
│  │   dot   = I[k]·I[k-1] + Q[k]·Q[k-1]                            │   │
│  │   ratio = cross / abs_dot  (int64 to prevent overflow!)          │   │
│  │   angle = ratio × (1 - 0.28125 × ratio²) + quadrant correction  │   │
│  │   output = angle × fm_scale >> 15                                │   │
│  │                                                                   │   │
│  │ fm_scale = 28000 / max_angle  (maps ±75kHz to 85% of Q15)       │   │
│  │ THD: < 0.3%                                                      │   │
│  │                                                                   │   │
│  │ Input:  int16 IQ pairs @ 256 kSPS                                │   │
│  │ Output: int16 MPX mono audio @ 256 kSPS                         │   │

 KEY BUGS FOUND AND FIXED
══════════════════════════════════════════════════════════════════════════

 1. Noise blanker threshold (avg>>8)*20 = 1860 blanked ALL samples
    at mag=24000 → output was constant IQ → discriminator = silence
    FIX: avg * (threshold+2) — proper magnitude-relative comparison

 2. FM discriminator (cross << 14) overflowed int32 for IQ > ±5000
    → garbage atan2 → single-polarity output
    FIX: ((int64_t)cross << 15) / abs_dot — 64-bit division

 3. WebSocket sent ~200 tiny PCM frames/sec → mbedTLS overload → drops
    FIX: Buffer 9600 samples (100-200ms) before flush → ~5-10 frames/sec

 4. Discriminator scale 16384/max_angle wasted 6dB of dynamic range
    FIX: 28000/max_angle → 85% Q15 utilization
